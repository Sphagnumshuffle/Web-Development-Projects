Why use CSS Grid

Well, that is an excellent question my friend! First of all CSS grid allows you to modify your lements in both vertically and horizontally. You can't do this with flexbox, so CSS Grid takes the lead here. But you don't have to count out either because they synergize pretty well. Flexbox has better browser support but I sincerely hope that CSS Grid gets that kind of browser support fast ( or maybe it already has)

I personally find that CSS Grid makes it easier to design your page layout. You can just split it into different sized boxes. And the great part here is that you can easily swap the order of the boxes, so if you want to change layout in the fly it is very much possible utilizing CSS Grid. CSS Grid also has some own tips and tricks to make your web pages responsible using media queries and CSS Grids own special element properties. But we well get there eventually if you manage to tag along for so long.

How do I use this great CSS Grid

Well it is pretty easy if you are familiar with the Flexbox. You just basically assign your parent element (usually class or id of it is named "container" or "wrapper") and tada! Congrats, you just created a simple CSS Grid. Here is an example how to put in use in css-file:

.container {
display: grid;
}

Sure this doesn't do pretty much anything if you did the previous step. You can alter how many columns and rows your CSS Grid container or wrapper includes. Here is a simple example where I split the page horizontally into two spaces; smaller for navigation element and bigger for the main content:

.container {
display: grid;
grid-template-columns: 1fr 4fr;
}

Here I just created a split where I emphatize that navigation takes 1 fraction of designated space and content takes 4 fractions. I made this because I want to show that the content is more important and should always take 4/5 fractions compared to navigations 1/5 fraction.

So there I split the page into two different portions horizontally. Now I need to do the same vertically. Lucky for us, it is very similar to the previous split (which was horizontal). So we just replace the word column with row:

.container {
display: grid;
grid-template-columns: 1fr 4fr;
grid-template-rows 100px 150px;
}

Easy as that! Next up; change the order of our grid items in container/wrapper. This one is also very handy feature of CSS Grid and it makes super easy to alter the layout of your page. You give items in your parent container or wrapper grid-area name which you use in one of great CSS properties. Here I'm going to stick with our example where I have to columns; navigation and main-section. Note that these are divs in my html and therefore I use dot-sign in front of them in examples. Here is how I point names to my items in container:

.navigation {
grid-area: navigation;
}

.content {
grid-area: content;
}

So how do I change to order of my items in layout? I'm going to show you my correct layout and opposite of it in the next example. This is my preferred layout order:

.container {
display: grid;
grid-template-columns: 1fr 4fr;
grid-template-areas:
"navigation content";
}

You can see that I used the grid-template-areas here to arrange the items as I see fit. Note that we used our previously defined grid-area values in the grid-template-areas property. Now if I wanted to change the order to opposite that would be easy to do, like this:

.container {
display: grid;
grid-template-columns: 1fr 4fr;
grid-template-areas:
"content navigation";
}

Here you can see that I just swapped the places of content and navigation in grid-template-areas and the layout is now completely different compared to the first and actually it is opposite. See how simple and easy that was!
